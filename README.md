# 矿场合伙共用账本（Supabase 版）

这是一个面向小团队的极简财务系统，目标是：公开透明、相互约束、流程不过度复杂。

## 已实现能力

- 成员登录（Supabase Auth）
- 管理员可新增业务成员，并绑定登录账号到成员
- 每人只能新增/删除自己录入的数据（RLS）
- 所有人可查看完整账目与凭证（透明）
- 机器算力占比自动计算
- 灵活结算周期（几天或一个月都可）
- 结算后锁账（锁定周期内记录不可改）
- 备用金池余额自动统计
- 每个周期自动计算：
  - 应承担费用
  - 已贡献金额（入金 + 个人垫付）
  - 净额（超出/结余）
  - 个人目标备用金与建议补款

## 账务口径

- 分摊比例：`个人占比 = 个人机器总算力 / 全体机器总算力`
- 公共支出：默认按占比分摊
- 入金记录：正数=入金，负数=提回
- 支出来源：
  - `pool` 备用金池支付
  - `personal` 个人垫付
- 结算净额：`净额 = 本期(入金 + 个人垫付) - 本期应承担`

## 快速启动

1. 创建 Supabase 项目。
2. 打开 Supabase SQL Editor，执行 `supabase/schema.sql`。
3. 在 Supabase Auth 中创建账号（邮箱+密码）。
4. 将你的账号设为管理员：

```sql
update public.profiles
set role = 'admin'
where id = '<该成员的 auth.users.id>';
```

5. 启动网站：

```powershell
node server.js
```

6. 打开：

```text
http://localhost:3000
```

7. 首次打开页面会提示输入：
   - Supabase URL
   - Supabase anon key

输入后会保存在浏览器 `localStorage`。

## 管理员初始化（V2）

1. 管理员登录后，在页面顶部可看到“成员与账号管理（仅管理员）”。
2. 先创建业务成员（例如：A合伙人、B合伙人）。
3. 在“账号邮箱 -> 绑定成员”中，把每个登录邮箱绑定到对应成员。
4. 在“成员算力占比”里维护每个成员机器总算力。
5. 普通账号登录后只能为自己绑定的成员录入数据。

## 使用建议（你们团队当前规则）

- 先维护好每个人机器总算力。
- 每次结算先建周期，再看自动结算表。
- 争议费用先备注说明，不要改历史记录。
- 周期确认后立即锁账，后续通过新记录更正。
